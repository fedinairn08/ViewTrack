# ViewTrack - Система управления просмотром фильмов

ViewTrack - это веб-приложение для управления личной коллекцией фильмов с возможностью отслеживания просмотренных фильмов, добавления отзывов и получения AI-рецензий

## 🎯 Основные функции

### Для пользователей:
- **Регистрация и авторизация** - безопасная система входа с JWT токенами
- **Управление коллекцией фильмов**:
  - Добавление фильмов в список "К просмотру"
  - Отметка фильмов как просмотренных
  - Удаление фильмов из списков
- **Система рецензий**:
  - Добавление отзывов на фильмы
  - Оценка фильмов по 10-балльной шкале
  - Редактирование и удаление своих отзывов
- **Внешние отзывы** - автоматическая загрузка отзывов о фильме из Кинопоиска
- **AI-рецензии** - автоматическая генерация рецензий с помощью GigaChat
- **Поиск и фильтрация**:
  - Просмотр всех фильмов в базе
  - Фильтрация по жанрам
  - Поиск по названию
- **Профиль пользователя** - управление личными данными и аватаром

### Для администраторов:
- **Управление фильмами**:
  - Добавление новых фильмов
  - Редактирование существующих фильмов
  - Удаление фильмов
  - Загрузка постеров
- **Управление жанрами**:
  - Создание новых жанров
  - Удаление жанров
- **Управление режиссерами**:
  - Добавление режиссеров
  - Редактирование информации о режиссерах
  - Загрузка фотографий режиссеров
  - Удаление режиссеров

## 🛠 Технологический стек

### Backend:

- **Spring Security** - аутентификация и авторизация

- **MinIO** - S3-совместимое хранилище для изображений

### AI интеграция:
- **GigaChat API** - генерация AI-рецензий на фильмы

### Инфраструктура:
- **Docker & Docker Compose** - контейнеризация
- **Gradle** - система сборки

## 🚀 Установка и запуск

### Предварительные требования:
- Java 17 или выше
- Docker и Docker Compose
- Git

### 1. Клонирование репозитория:
```bash
git clone <repository-url>
cd viewTrack
```

### 2. Настройка переменных окружения:
Создайте файл `application-dev.properties` в корне проекта:
```properties
# База данных
DB_URL=jdbc:postgresql://localhost:5433/viewTrack
DB_USERNAME=root
DB_PASSWORD=root

# JWT токены
ACCESS_JWT_TOKEN_AGE=86400
REFRESH_JWT_TOKEN_AGE=604800

# MinIO (S3-совместимое хранилище)
MINIO_ENDPOINT=http://localhost:9000
MINIO_ACCESS_KEY=root
MINIO_SECRET_KEY=minio123

# GigaChat API
GIGACHAT_CLIENT_ID=your_gigachat_id_here
GIGACHAT_CLIENT_SECRET=your_gigachat_secret_here

# poiskkino.dev API (опционально, для импорта внешних отзывов с Кинопоиска)
KINOPOISK_API_KEY=your_kinopoisk_api_key_here
```

### 3. Запуск инфраструктуры:
```bash
docker-compose up -d
```

Это запустит:
- PostgreSQL на порту 5433
- MinIO на портах 9000 (API) и 9090 (консоль)

### 4. Сборка и запуск приложения:
```bash
./gradlew bootRun
```

Приложение будет доступно по адресу: `http://localhost:8080`

### 5. Доступ к интерфейсам:
- **Основное приложение**: http://localhost:8080
- **Swagger UI**: http://localhost:8080/swagger-ui/index.html#
- **MinIO консоль**: http://localhost:9090 (root/minio123)

## 🔧 Конфигурация

### Настройка GigaChat:
1. Получите API ключ от GigaChat
2. Добавьте его в переменную `GIGACHAT_CLIENT_SECRET` в `application-dev.properties`

### Настройка Кинопоиска (poiskkino.dev):
1. Получите API ключ через бота [@poiskkinodev_bot](https://t.me/poiskkinodev_bot)
2. Добавьте его в переменную `KINOPOISK_API_KEY` в `application-dev.properties`
3. После этого на странице фильма появится блок с внешними отзывами, если они есть в источнике

### Настройка MinIO:
1. MinIO запускается автоматически через Docker Compose
2. Создайте bucket `view-track` в MinIO консоли
3. Настройте права доступа для приложения

## 🏗 Архитектура

Приложение построено по принципам:
- **MVC (Model-View-Controller)** - разделение логики
- **REST API** - для взаимодействия с клиентом
- **Service Layer** - бизнес-логика
- **Repository Pattern** - работа с данными
- **DTO Pattern** - передача данных между слоями
